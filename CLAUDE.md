# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Personal academic portfolio website for Saad Hossain, hosted on GitHub Pages at `saad-hossain.github.io`.

## Development

### Setup

```bash
npm install
```

### Build

```bash
npm run build
```

This generates HTML files in the root directory from templates + JSON data.

### Workflow

1. Edit content in `data/*.json` files (experiences, publications, news, etc.)
2. Edit templates in `templates/*.html` if layout changes are needed
3. Run `npm run build` to regenerate HTML
4. Open generated HTML files in browser to preview
5. Push to `main` branch to deploy via GitHub Pages

## Architecture

### Data Files (`data/`)

- `profile.json` - Name, bio, social links
- `news.json` - Homepage news items
- `experience.json` - Professional and research experience (with bullet points)
- `publications.json` - Publications and talks
- `navigation.json` - Navigation menu items

### Templates (`templates/`)

- `partials/head.html` - Common `<head>` content (fonts, CSS links)
- `partials/navbar.html` - Shared navbar component
- `index.html`, `experience.html`, `publications.html`, `cv.html` - Page templates

Template syntax:

- `{{variable}}` - Insert value
- `{{#each array}}...{{/each}}` - Loop over array
- `{{#if condition}}...{{/if}}` - Conditional
- `{{> partial}}` - Include partial
- `{{this}}` - Current item in primitive array loop

### CSS (`css/`)

- `base.css` - CSS variables, reset, typography, navbar
- `components.css` - Reusable components (cards, timeline, ribbons, buttons)
- `pages/*.css` - Page-specific styles

### Generated Files (root)

- `index.html`, `experience.html`, `publications.html`, `cv.html` - Generated by build script, do not edit directly

### Build System

- `src/build.ts` - TypeScript build script with simple template engine
- `package.json` - npm scripts and dependencies
- `tsconfig.json` - TypeScript configuration

## Layout Constraints (About/Index Page)

These alignment rules are **unchanging** and must be preserved during any layout adjustments:

1. **Sidebar bottom aligns with image bottom**: The sidebar's bottom edge must always align exactly with the bottom of the profile image (not the icons below it). The sidebar uses `position: fixed` with `transform: translateY(-100%)` so `--sidebar-top` controls where the bottom lands.

2. **Name divider line anchored to image top**: The border-bottom on `.about-name` must sit at the same y-level as the top of the profile image. This means `.about-name` has `margin-bottom: 0`. NEVER add margin-bottom to `.about-name`.

3. **Coupled movement via shared CSS variables**: Both alignments are locked by shared CSS custom properties in `:root` of `base.css`:
   - `--index-content-top` — content padding-top on index page
   - `--profile-pic-size` — image width/height
   - `--h1-font-size`, `--h1-line-height`, `--about-name-pad-bottom` — name element dimensions
   - `--sidebar-top` — computed from the above: `content-top + name-height + pic-size`

   To move the content up/down, change `--index-content-top`. To resize the image, change `--profile-pic-size`. The sidebar automatically follows. **Never use hardcoded values for these dimensions.**

## Adding Content

### Add a new experience

Edit `data/experience.json`, add to `professional` or `research` array:

```json
{
  "title": "Job Title",
  "company": "Company Name",
  "location": "City, State",
  "startDate": "Jan 2024",
  "endDate": "Present",
  "logo": "images/company-logo.png",
  "summary": "Brief description",
  "bullets": ["Achievement 1", "Achievement 2"]
}
```

### Add a new publication

Edit `data/publications.json`, add to `publications` array:

```json
{
  "title": "Paper Title",
  "authors": "A. Author, <u>S. Hossain</u>, B. Coauthor",
  "venue": "Conference Name 2024",
  "thumbnail": "images/paper-thumbnail.png",
  "links": [{ "label": "PAPER", "url": "https://..." }]
}
```

### Add a new page

1. Add entry to `data/navigation.json`
2. Create `templates/newpage.html` template
3. Create `css/pages/newpage.css` if needed
4. Add page config to `pages` array in `src/build.ts`
5. Run `npm run build`
